# The Industrialists — Founder Profiles

A [founderprofiles.ai](https://www.founderprofiles.ai/)-style archive of ~80 American
industrialists. Each profile is **researched from the live web with citations** and can
be **narrated aloud** with ElevenLabs.

- **Stack:** Next.js 16 (App Router) · Tailwind v4 · TypeScript · Vercel
- **Profiles:** generated by Claude (`claude-opus-4-8`) + server-side web search, with
  inline `[n]` citations tied to a numbered sources list
- **Narration:** on-demand ElevenLabs TTS, cached (in-memory + optional Vercel Blob),
  with the browser caching the audio for instant replay

## Quick start

```bash
npm install
cp .env.example .env.local   # fill in keys
npm run dev                  # http://localhost:3000
```

The homepage lists everyone grouped by sector; click a name for the profile + a
"Listen" button. Two profiles (Henry Ford, William Levitt) ship hand-written so the
UI is real before you generate the rest.

## Generate profiles (deep research + citations)

Requires `ANTHROPIC_API_KEY`.

```bash
node scripts/generate-profiles.mjs --limit 3     # try a few first, eyeball quality
node scripts/generate-profiles.mjs               # generate all missing
node scripts/generate-profiles.mjs --only ford   # one person
node scripts/generate-profiles.mjs --force       # regenerate existing
node scripts/generate-profiles.mjs --model claude-sonnet-4-6   # cost-down lever
```

Output lands in `data/profiles/<slug>.json` and is read at build time — **commit it**
so the deployed site needs no Anthropic key.

Each run does two Claude calls per person: a web-search research turn that builds a
sourced dossier, then a structuring turn that emits the profile JSON with `[n]`
citations. (Split intentionally: web-search citations can't be combined with structured
outputs in one call.)

### Depth standard (no Wikipedia rewrites)

Profiles are written to an archival standard, not summarized from encyclopedias. The
research prompt explicitly hunts **definitive biographies and company histories**,
**newspaper/periodical archives** (NYT, WSJ, Time, Fortune, trade press, with dates),
and **primary/archive sources** (memoirs, letters, court records, museum holdings,
HBS cases). Wikipedia/Britannica are allowed only as an index to find better sources.
Each profile carries:

- **`sources[]`** — typed citations (`book` · `newspaper` · `journal` · `archive` ·
  `interview` · `web`) with `author`, `publisher`, `year`, and `pages` where known;
  `url` is optional (books/archives often have none).
- **`excerpts[]`** — 2–4 *verbatim* passages from books/newspapers, each with an
  attribution and a `sourceId` link.
- **`furtherReading[]`** — the canonical books on the subject (author, title, year, note).

`data/profiles/henry-ford.json` is the hand-written **gold-standard template** — the
shape, citation style, and depth every generated profile is held to.

The `Profile`/`Source` shapes live in `lib/types.ts`; new fields are optional so older
profiles still render.

## Bulk generation without an Anthropic key (agent workflow)

`scripts/deep-profiles.workflow.mjs` is a Claude Code **Workflow** that fans out one
research agent per industrialist. Each agent uses live `WebSearch`/`WebFetch`, reads the
Ford template for format, writes its own `data/profiles/<slug>.json`, and self-validates
the JSON. The people list is inlined in the script (everyone except Ford), so it runs
with no `ANTHROPIC_API_KEY` — useful when only the placeholder key is present. Launch it
from Claude Code with the `Workflow` tool pointed at that script; progress shows under
`/workflows`.

## Narration

Set `ELEVENLABS_API_KEY` (and optionally `ELEVENLABS_VOICE_ID` / `ELEVENLABS_MODEL_ID`).
`POST /api/narrate { slug }` returns `audio/mpeg`. The browser caches the audio so
replays are instant; the server caches per warm instance, and durably in Vercel Blob
when `BLOB_READ_WRITE_TOKEN` is present.

## Deploy

```bash
vercel --prod
```

Set `ELEVENLABS_API_KEY` (and any optional vars) in the Vercel dashboard. Add a Vercel
Blob store for durable narration caching. `ANTHROPIC_API_KEY` is **not** needed in
production — profiles are pre-generated and committed.

## Layout

```
app/                 homepage, [slug] profile page, api/narrate route
components/           ProfileCard, NarrationPlayer
data/industrialists.json   the seed list (from the source PDF)
data/profiles/*.json       generated profiles
lib/                 profiles loader, types, ElevenLabs helper
scripts/generate-profiles.mjs   the deep-research generator
```
